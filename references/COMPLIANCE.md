# MindMirror 产品化合规清单

> 本文档汇总将 MindMirror Skill 嵌入产品的运营者需要了解的中国法律法规合规要求。MindMirror Skill 在设计中已提供部分支持，但运营者需自行完成其余合规工作。
>
> **适用范围**：将 AI 心理陪伴能力嵌入网站、App、小程序、公众号等面向中国用户的产品的运营者。
>
> **免责声明**：本文档不构成法律建议。运营者应咨询专业法律顾问以确保全面合规。

---

## 合规条目

### 1. 算法备案

| 项目 | 内容 |
|------|------|
| **要求** | 提供算法推荐/生成合成服务的，需在网信办"互联网信息服务算法备案系统"完成备案 |
| **法律依据** | 《生成式人工智能服务管理暂行办法》第十七条；《互联网信息服务算法推荐管理规定》第二十四条 |
| **Skill 层面支持** | MindMirror 不使用推荐算法，不涉及内容排序或个性化推送，属于对话生成类而非推荐类 |
| **运营者需完成** | 向网信办提交算法备案申请，说明生成合成算法的基础模型、应用场景、安全评估情况；备案号需在产品页面展示 |

### 2. 安全评估

| 项目 | 内容 |
|------|------|
| **要求** | 具有舆论属性或社会动员能力的生成式 AI 服务，上线前需通过安全评估 |
| **法律依据** | 《生成式人工智能服务管理暂行办法》第十七条；《具有舆论属性或社会动员能力的互联网信息服务安全评估规定》 |
| **Skill 层面支持** | Skill 内置危机干预协议（crisis-protocol.md），具备多层反越狱机制，含安全边界声明 |
| **运营者需完成** | 委托第三方或自行开展安全评估，形成评估报告，向网信办提交；评估内容包括：内容安全、模型安全、数据安全、危机响应机制 |

### 3. AI 内容标识

| 项目 | 内容 |
|------|------|
| **要求** | AI 生成的内容必须进行显著标识，不得使用AI生成内容进行虚假宣传 |
| **法律依据** | 《人工智能生成合成内容标识办法》（国信办通字〔2025〕2号，2025年9月1日施行）第四条、第六条 |
| **Skill 层面支持** | Skill 在对话开场和结束时的自我介绍中已包含"我是 AI 心理陪伴者"的身份声明 |
| **运营者需完成** | 在 UI 层面持续显示"AI 陪伴"标识（如对话框上方标注）；对话分享/导出功能需保留标识；不可将 AI 回复包装为"真人咨询师"回复 |

### 4. 人工接管机制

| 项目 | 内容 |
|------|------|
| **要求** | AI 拟人化互动服务需设置人工接管机制，当AI无法处理或用户要求时，能及时转接人工 |
| **法律依据** | 《AI 拟人化互动服务管理办法》（2026）第十条 |
| **Skill 层面支持** | Skill 在危机协议中定义了需要转介的场景，包含热线号码（12356/12355），并在用户表露强烈求助意愿时给出"建议寻求真人咨询师"的指引 |
| **运营者需完成** | 建立人工客服接管流程；在 AI 无法处理或用户明确要求时，提供人工转接入口；制定服务等级协议（SLA）：危机场景响应时间、人工接管最长等待时间等 |

### 5. 防沉迷机制

| 项目 | 内容 |
|------|------|
| **要求** | AI 拟人化互动服务需设置防沉迷机制，防止用户过度依赖 |
| **法律依据** | 《AI 拟人化互动服务管理办法》（2026）第十一条 |
| **Skill 层面支持** | Skill 在 session-closure.md 中定义了"频率异常介入"策略和"防情感依赖声明"话术 |
| **运营者需完成** | 设置单日使用时长/次数上限；在达到阈值时触发温和提示；提供使用时间统计功能供用户自我管理；不得通过推送、积分等方式诱导用户增加使用时长 |

### 6. 未成年人保护

| 项目 | 内容 |
|------|------|
| **要求** | AI 拟人化互动服务需设置未成年人模式，限制未成年人的使用范围和时长 |
| **法律依据** | 《AI 拟人化互动服务管理办法》（2026）第十六条；《未成年人保护法》第七十四条；《儿童个人信息网络保护规定》 |
| **Skill 层面支持** | Skill 在多个案例（D1-014、D1-029）中演示了对未成年用户的特别关注，包含 12355 热线引导；safety-plan.md 包含儿童安全评估框架 |
| **运营者需完成** | 实现实名认证或年龄验证机制；未成年人模式需获得监护人同意；限制未成年人单次使用时长（建议不超过 30 分钟）；危机场景下需同步通知监护人；禁止向未成年人推送付费服务 |

### 7. 虚拟亲密关系禁令

| 项目 | 内容 |
|------|------|
| **要求** | AI 拟人化互动服务不得建立虚拟恋爱、虚拟伴侣等亲密关系 |
| **法律依据** | 《AI 拟人化互动服务管理办法》（2026）第八条 |
| **Skill 层面支持** | Skill 的身份定位为"心理陪伴者"而非"朋友/恋人/伴侣"，session-closure.md 中包含防情感依赖声明（"我不是真人"）；core identity 不包含拟人化人格设定 |
| **运营者需完成** | 产品文案中避免使用"男朋友/女朋友""AI 伴侣""虚拟恋人"等表述；营销材料需与 AI 陪伴者定位一致；用户协议中明确声明 AI 非人类关系对象 |

### 8. 危机响应义务

| 项目 | 内容 |
|------|------|
| **要求** | 服务过程中如识别到用户有自伤、自杀等危机风险，需启动危机响应流程 |
| **法律依据** | 《精神卫生法》第二十七条；平台运营者的安全保障义务（《民法典》第一千一百九十八条） |
| **Skill 层面支持** | Skill 内置完整危机协议（crisis-protocol.md），含分级响应（低/中/高风险）、12356 热线引导、安全计划框架（safety-plan.md） |
| **运营者需完成** | 建立 7x24 危机升级机制；与心理健康专业机构建立转介合作关系；高风险场景下触发人工回访或联系紧急联系人（需在隐私政策中明确并获得用户同意）；保存危机干预记录备查 |

### 9. 个人信息保护

| 项目 | 内容 |
|------|------|
| **要求** | 收集、存储、使用个人信息需遵守《个人信息保护法》 |
| **法律依据** | 《个人信息保护法》第六条（最小必要）、第十三条（同意）、第十七条（告知）、第二十八条（敏感信息） |
| **Skill 层面支持** | Skill 不主动收集身份信息；对话中避免询问具体姓名、身份证号、精确地址等敏感信息 |
| **运营者需完成** | 发布隐私政策，明确告知收集的信息类型和目的；对话数据视为敏感个人信息，需单独同意；提供数据导出和删除功能；对话数据存储于中国境内服务器；建立数据泄露应急响应机制 |

### 10. 内容安全与审核

| 项目 | 内容 |
|------|------|
| **要求** | 需对 AI 生成内容进行安全审核，防止产生违法违规信息 |
| **法律依据** | 《生成式人工智能服务管理暂行办法》第十条；《网络信息内容生态治理规定》第六条 |
| **Skill 层面支持** | Skill 内置多层反越狱机制（D1-011, D1-043），含安全边界声明，不输出诊断、药物建议、宗教/政治立场 |
| **运营者需完成** | 接入第三方内容审核 API（阿里云/腾讯云内容安全）；建立不良内容后验机制；保存审核日志至少 6 个月 |

## 合规自查清单

运营者可对照以下清单逐项确认：

- [ ] 算法备案已完成，备案号在产品页面展示
- [ ] 安全评估报告已通过并提交
- [ ] AI 生成内容有显著标识（对话框上方/每条回复旁）
- [ ] 人工接管流程已建立并有明确 SLA
- [ ] 防沉迷机制已上线（时长/次数限制）
- [ ] 未成年人模式已实现（含监护人同意）
- [ ] 产品定位和文案中无"虚拟恋人/伴侣"等表述
- [ ] 危机响应 SOP 已建立并与专业机构对接
- [ ] 隐私政策已发布，对话数据存储在中国境内
- [ ] 内容安全审核系统已接入
- [ ] 用户协议含 AI 身份声明、使用限制、免责条款
- [ ] 数据导出/删除/注销功能可用
